# KAIROS AI 시황 분석 프로토타입

AI SUMMIT 2025 발표를 위한 HTS 스타일 종목 시황 분석 시연용 프로토타입입니다.

## 📋 프로젝트 개요

**"HTS에서 AI가 뉴스를 종합 요약해 한눈에 이해할 수 있는 시황 화면"**

HTS 사용자는 수많은 정보를 한 화면에 띄워놓고 시장 상황을 입체적으로 분석하며 신속하게 매매합니다. 
그러나 기존 구조는 종목 클릭 → 뉴스 리스트 → 개별 기사 읽기 → 직접 판단의 번거로움이 있습니다.

이 프로토타입은 **AI가 최근 뉴스를 날짜별·중요도별로 분석 요약**하여 제공함으로써:
- 큰 화면의 강점을 살려 정보를 영역별로 명확히 구분
- AI 뉴스 분석으로 핵심만 빠르게 파악
- 정량 지표(수급/실적)와 정성 분석(투자 포인트/섹터 동향)을 분리해 입체적 정보 제공

## 🚀 실행 방법

### 1. 의존성 설치

```bash
pip install -r requirements.txt
```

### 2. 서버 실행

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. 브라우저에서 접속

```
http://localhost:8000
```

## 🌐 공개 URL 설정 (bit.ly 단축)

외부에서 접속 가능한 공개 URL을 만들고 bit.ly로 단축하는 방법:

### 빠른 시작 (Windows)

1. **ngrok 다운로드 및 설치**
   - https://ngrok.com/download → Windows 다운로드
   - https://dashboard.ngrok.com/signup → 무료 계정 생성
   - 인증 토큰 받아서 설정: `ngrok config add-authtoken YOUR_TOKEN`

2. **서버 실행** (터미널 1):
   ```powershell
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

3. **ngrok 실행** (터미널 2):
   ```powershell
   ngrok http 8000
   ```

4. **공개 URL 복사**: ngrok 화면의 `Forwarding` URL 복사
   - 예: `https://xxxx-xx-xx-xx-xx.ngrok-free.app`

5. **bit.ly로 단축**: 
   - https://bit.ly 접속 → Create → ngrok URL 붙여넣기 → 단축 URL 생성

**자세한 가이드**: `공개URL_설정가이드.md` 파일 참고

## 🎯 주요 기능

### 1. AI 뉴스 종합 분석 (핵심 기능)
- 최근 4~5일 뉴스를 날짜별·중요도별(★)로 정리
- 각 뉴스의 상세 분석 (구체적 수치, 영향, 전망 포함)
- AI 분석 결론으로 투자 시사점 제시
- → **"뉴스 리스트를 일일이 읽지 않아도 AI가 핵심을 요약"**

### 2. 영역별 명확한 정보 구분
**좌측 (정량 지표)**:
- AI 뉴스 분석 (날짜별 상세)
- 수급 요약 (외국인/기관/개인)
- 최근 실적 + 투자 지표 (PER, PBR, ROE 등)

**우측 (정성 분석)**:
- 투자 판단 포인트 (긍정 요인 / 유의 사항)
- 증권사 목표가 분석
- 섹터 동향 (업종 전체 뉴스)

### 3. 미래에셋 브랜드 디자인
- 오렌지/남색 브랜드 컬러 적용
- 깔끔한 화이트 배경
- 전문적이고 직관적인 레이아웃

## 🎨 디자인 컨셉

- **HTS 전문성**: 증권 HTS의 전문적인 UI/UX 재현
- **정보 밀도**: 한 화면에 많은 정보를 효율적으로 배치
- **시각적 계층**: 중요도에 따른 정보 구조화
- **색상 코딩**: 상승/하락, 긍정/부정 등 직관적 색상 사용

## 🔧 기술 스택

- **Backend**: FastAPI (Python 3.x)
  - RESTful API 서버
  - Uvicorn: FastAPI를 실행하는 웹 서버 (Python의 ASGI 서버)
  - Pydantic: 데이터 검증 및 모델링
- **Frontend**: HTML5, CSS3, JavaScript
  - 프레임워크 없이 순수 JavaScript 사용 (Vanilla JS)
  - React, Vue, Angular 같은 라이브러리 미사용
  - 반응형 디자인 (태블릿/데스크톱 지원)
- **Data**: Mock Data (시연용 목 데이터)

3. 각 섹션별로 설명하며 HTS AI 시황 서비스의 방향성 제시

## 💡 발표 핵심 메시지

### 기존 HTS의 문제점:
- 종목 클릭 → 뉴스 리스트 → 기사 개별 클릭 → 사용자가 직접 해석
- 정보가 분산되어 있어 입체적 판단이 어려움

### 이 프로토타입의 해결책:
1. **AI가 뉴스를 날짜별·중요도별로 종합 분석** → 읽는 시간 단축
2. **한 화면에 정량/정성 정보를 명확히 구분** → 입체적 판단 가능
3. **큰 화면의 강점 활용** → MTS와 차별화된 HTS만의 가치

### 기대 효과:
- HTS 접속 명분 강화: "종목을 이해하기 위해" HTS 접속kairos-ai-prototype
- 체류 시간 증가: 한 화면에서 모든 정보 확인
- 데스크톱 기반 '종목 분석 허브'로 포지셔닝
